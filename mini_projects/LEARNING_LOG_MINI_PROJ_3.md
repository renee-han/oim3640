## Date: 2026

**What I'm building:** 
[For mini project 3, I'll be building a web app through Flask that takes in one's address or current location based on longitude and latitude and assists in finding the nearest MBTA train station.]

**Why I chose this:** 
[This project is important because it is an extremely functional and accessible tool for people who are trying to conveniently and efficiently find the nearest MBTA stop. Oftentimes when using other map sources i.e. Google Maps, there can be an overwhelming amount of MBTA suggestions that pop up. This tool eases that decision fatigue and can especially support newcomers to Boston like tourists.]

**Core features:** 
[Functions that take in long and lat coordinates, connects with both API sources i.e. MBTA and MapBox, returns the closest station, and ideally, displays all this on a map. It should also be able to handle errors neatly i.e. if a user inputs a location that isn't covered by MBTA routes (outside of MA) or if there's an error with the API itself i.e. reached max tries.]

**What I don't know yet:** 
[How to seamlessley unite both API's i.e. have them 'talk to one another' and be able to have a functionality piece that takes in a user location and returns the closest MBTA station.]
